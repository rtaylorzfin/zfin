- [ ] JIRA TICKET: multi-arch STOCK images on ghcr.io.
      The stock images (ghcr.io/zfin/zfin-{base,compile,db,solr,httpd,tomcat,...}:${ZFIN_RELEASE})
      are single-arch today — nothing in CI (.github/workflows/{ant,eslint}.yml) builds/pushes them
      with buildx, so whatever is on ghcr.io is whatever host last pushed (almost certainly amd64).
      On an arm64 Mac, `FROM ghcr.io/zfin/zfin-db:main` then pulls amd64 and runs under emulation.
      Fix: publish a manifest list under the plain tag (:main) so `docker pull ...:main` auto-selects
      arm64/amd64 and Dockerfile FROM lines never change. Either `buildx --platform
      linux/amd64,linux/arm64 --push`, or build each arch natively as arch-suffixed tags
      (:main-arm64 / :main-amd64 — an accepted pattern) and combine with `buildx imagetools create`.
      NOTE: STOCK family only. The PRELOADED images are deliberately local-only (real data, never
      pushed), so they need only the builder's native arch — build-preloaded.groovy handles that
      with plain `docker build`.
